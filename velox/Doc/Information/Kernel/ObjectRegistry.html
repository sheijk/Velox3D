The Object Registry

The object registry is a singleton object which maintains a mapping between managed objects and unique keys. What it does:
- objects can be registered and unregistered
- the key of a known object can be retrieved
- an object can be retrieved using a known key
- a new unique key can be generated
- objects can specify the key they are registed with
- keys can be converted to and from std::string

Objects that shall be registered at the registry need to be derived from the class VNamedObject. Object keys are of type VObjectKey.
