Velox Editor

  1. Implementierung des Editor Frameworks:
    • Laden von Texturen in jpg im Texture Frame
    • Zugriff auf Fenster des Gfx Services
    • Erstellung des Grid Frames und Bearbeitungsfunktionen für Brushes
    • Import Modul für 3ds Dateien
    • Entwicklung eines Plugin Systems für den Editor
    • Serializer für Konfigurationseinstellung und Speicherung der Szene im XML Format
    • Erstellung eines Scenegraph Tree und Bearbeitungsfunktionen innerhalb der Scene View
  2. Plugins:
    • Geometrie Compiler
      • Portal Compiler
      • Bsp Tree Compiler (additional)
      • Outdoor Quadtree Compiler (additional)
    • Light & Radioscity Calculator
    • Animation Controlling
  3. Aufbau:
    • Grid Frame - Die Benutzeroberfläche für die Erstellung der Level Geometrie
    • Editor Frame: - Stellt die erstellte Geometrie (vorwiegend Brushes) aus dem Scenegraph dar
    • Texture Frame: - Stellt Texturen als kleines Thubnail dar. Einzelne Texturen können via Drag & Drop auf die Geometrie im Editor Frame gezogen werden
    • Scene Frame: - Zeigt ein Abbild des aktuellen Scenegraphs und bietet Bearbeitungsmöglichkeiten
    • Log Frame: - Text Ausgabe für Logging Zwecke
    • Gfx Window: - Wird je nach Gebrauch eingeblendet und zeigt den aktuellen Scenegraph. Separate Ausführungsmöglichkeiten wie Gravity, Game Logik, Collision Detecting werden verfügbar sein.
  4. Features:
    • Fenster sind frei anzuordnen
    • Hotkeys konfigurierbar um Frames einzublenden
    • Szene kann vom Editor durch die Engine "realtime" gezeigt werden
    • Geometrie kann nach XML konvertiert werden und ist somit frei austauschbar. Entwickler können bestimmt Elemente speichern und in verschiedenen Szenen einbinden
